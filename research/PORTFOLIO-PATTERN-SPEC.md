# Portfolio Pattern Spec — the one-prompt blueprint

Maintained by the org-scribe agent. Purpose: this doc + the `portfolio` data array = enough to generate ANOTHER portfolio skin (Netflix, etc.) from a single prompt. System rules first (skin-agnostic), Spotify skin choices second. Every rule dated; Miles-sourced rules quoted.

## 1. The system (skin-agnostic)

1. **Two axes, never merged (Jul 4):** work units (projects/events — WHERE/WHEN) vs specialties (skills — WHAT). Brand-name work units carry credibility; specialty lenses answer "what would I hire him for." Alignment happens through shared verb vocabulary + cross-links, never by renaming one axis into the other.
2. **Content = real media + real numbers.** Every claim sits next to playable proof (proof adjacency). All on-screen stats DERIVED in code from the data array, never hand-typed. Miles Jul 4: expectation checksum ~32.3M plays / 100+ videos once complete; derived totals must converge to it, never be typed to it.
3. **Specialty lens anatomy (Jul 4, recruiter-lens verified):** eyebrow + title + meta line (events · years) → primary action + specialty story paragraph (Miles's words, moved high per his call) → organization chips (album sections + counts, tap-to-jump) → numbered proof list grouped under ALBUMS → prev/next specialty previews (never a dead end).
4. **Album rule (Miles, Jul 4):** proof rows group under the body of work they belong to — the SERIES/SEASON if one exists ("Be You · Season 1"), else the EVENT. Quarters rejected (corporate-calendar, not native to media metaphors).
5. **Proof rows:** thumb · title · derived sub-line (@handle · Mon YYYY — zero new claims) · one metric right-aligned. Rows expand and play IN PLACE (Miles: "just expands", no navigation jumps); one mounted video at a time; per-row description below the video (published caption lines or Miles's words only) + per-row source link.
6. **Naming nomenclature (Miles, Jul 4):** employee-comms reels = "{Full Name} Employee Spotlight"; full names come from published captions ("Meet {Name}…"), never guessed. Event reels may carry event prefix (his pattern: "MAX ’25 “Coolest Job” @Adobe | Firefly Feature").
7. **Copy gate:** everything user-visible passes COPY-PRINCIPLES.md (7 checks + resume/LinkedIn alignment). No em dashes. Numbers exact or absent. Miles's words win and are never overwritten, only appended-to with labeled alternates.
8. **Privacy wall:** the repo is PUBLIC. Internal source material (budgets, Workfront record names, approver names, briefs-as-written) NEVER enters the repo; working docs live in ~/Downloads/Claude/workfront-ingest/. Budget = Miles's private interview prep only.
9. **Work-unit headers** carry: role line ("{role} by Miles Spearman", résumé-grade, Miles-verified) + derived posting window (min–max reel dates in code) + [pending] one brief line + external partner credit from project records, gate-passed.
10. **Personal/creator work is present but fenced** ("Off the Clock" library) — proof of creator fluency, one line in the professional story, never mixed into work-section copy.
